# CLAUDE.md

Guidance for working in this repo. Keep it current as the project evolves.

## What this is

A single-purpose **Wear OS** app for the **Galaxy Watch 4 (55mm, round dial, Wear OS 3)**.
One full-screen circular **UNLOCK DOOR** button: tap → POST to an unlock API →
flash the screen **green ×3** on success, **red ×3** on failure. Also exposed as a
**Tile** for quick access from the watch face.

- UI: Jetpack Compose for Wear OS
- Language: Kotlin
- `minSdk 30` (Wear OS 3), `targetSdk 34`, `compileSdk 35`

## Layout

```
app/src/main/java/com/example/hellowatch/
  MainActivity.kt        # the screen, button, network call, flash animation
  UnlockTileService.kt   # the Tile (ProtoLayout) that launches MainActivity
app/src/main/AndroidManifest.xml   # activity + tile service + INTERNET permission
app/src/main/res/mipmap-*/         # generated app/tile icons (blue circle + text)
app/build.gradle.kts               # dependencies
.devcontainer/                     # Codespaces Android SDK + Gradle wrapper setup
```

> The package/namespace is `com.example.hellowatch` for historical reasons
> (started as a Hello World). The user-facing label is **"Unlock Door"** — don't
> confuse the two. Renaming the package is a larger refactor; avoid unless asked.

## How it actually works (non-obvious)

- **Tile → Activity handoff.** The Tile can't animate, so tapping it launches
  `MainActivity` with the `EXTRA_AUTO_UNLOCK` (`"auto_unlock"`) intent extra. In that
  mode the activity fires the unlock on first composition (`LaunchedEffect`), flashes
  the result, then `finish()`es back to the tile. Without the extra (launched from the
  app menu) the button works manually. Keep both paths working.
- **The unlock endpoint and headers** (URL, `User-Agent`, `Referer`) live as constants
  in `MainActivity.kt` and were copied from a browser version. If the API moves, update
  `UNLOCK_URL` there.
- **Cleartext:** the endpoint is HTTPS, so no network-security-config is needed. If it
  ever becomes plain `http://`, you must add a cleartext exception or the call fails
  silently (caught → red flash).

## Building (IMPORTANT)

There is **no local Android SDK** on the dev machine and **no `gradlew` checked in** —
the wrapper is generated by `.devcontainer/setup.sh`. You **cannot compile or run from
here**. Builds happen in a **GitHub Codespace**:

```bash
./gradlew assembleDebug      # APK → app/build/outputs/apk/debug/app-debug.apk
```

When you change code, you can't verify it compiles locally — say so, and ask the user
to build in the Codespace. The Tiles/ProtoLayout builder API is verbose and
version-sensitive; it's the most likely thing to break a build.

## Icons

App/tile icons are generated PNGs (blue `#007BFF` circle + white bold "UNLOCK DOOR"),
not vector drawables (VectorDrawable can't render text). They were produced with a
Pillow script using `/System/Library/Fonts/Supplemental/Arial Bold.ttf` (macOS path).
To regenerate, supersample 4× then downscale, and write both `ic_launcher.png` and
`ic_launcher_round.png` into every `mipmap-*` density bucket.

## Conventions

- Match the surrounding Compose/Kotlin style; keep the app single-screen and minimal.
- Colors are defined once as top-level `val`s in `MainActivity.kt` (`ButtonBlue`,
  `FlashGreen`, `FlashRed`, `ScreenBlack`) — reuse them.
- Commit only when asked. The default branch is `main` and commits are pushed there.
- macOS screenshot filenames contain a Unicode narrow no-break space (U+202F) before
  "PM" — a literal-space `git`/shell match will miss them; use a glob.
